# Структура messages в calc1

## Проблема

После анализа `E:\petProject\calc1\messages\en.json` выяснилось, что:

1. **В `categories` нет данных отдельных калькуляторов**
   - `categories.finance.subcategories.loans` - это строка "Loans" (название подкатегории)
   - `categories.finance.seo.sections.loans` - это объект с `name` и `description` (метаданные подкатегории)

2. **Структура messages/en.json:**
   ```
   {
     "categories": {
       "finance": {
         "title": "...",
         "description": "...",
         "subcategories": {
           "loans": "Loans",  // строка, не массив!
           "investments": "Investments",
           ...
         },
         "seo": {
           "sections": {
             "loans": {
               "name": "Loans",
               "description": "..."
             }
           }
         }
       }
     }
   }
   ```

## Где могут быть переводы калькуляторов?

1. **В компонентах** - возможно, переводы встроены в компоненты калькуляторов
2. **В отдельных файлах** - возможно, есть отдельные JSON файлы для каждого калькулятора
3. **Через namespace** - скрипт импорта ищет `namespace` в page.tsx, возможно, данные загружаются динамически
4. **В других локалях** - возможно, структура отличается в других языках
